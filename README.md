# Motricidad en Adultos mayores 

# Proyecto: Plataforma Web para la Estimulación Cognitiva y Psicomotriz en Adultos Mayores

# 1. Resumen Ejecutivo

Este proyecto propone el desarrollo de una plataforma web diseñada para ayudar a fortalecer las habilidades cognitivas y psicomotrices de los adultos mayores mediante ejercicios interactivos. Además de estimular la memoria, la atención y la coordinación, el sistema almacenará los resultados de cada sesión para que familiares o cuidadores puedan consultar el progreso del usuario.

La plataforma está pensada para desarrollarse en un periodo de tres semanas, utilizando tecnologías web sencillas y de fácil implementación.

---

# 2. Definición del Problema

Con el paso de los años, muchas personas adultas mayores presentan disminución en la memoria, atención, coordinación y velocidad de reacción.

Actualmente existen pocas herramientas accesibles que permitan ejercitar estas capacidades y registrar automáticamente el desempeño del usuario.

Por ello se propone una página web que ofrezca ejercicios interactivos y almacene los resultados para dar seguimiento a la evolución de cada persona.

Desarrollar una plataforma web que permita estimular y evaluar habilidades cognitivas y psicomotrices mediante ejercicios interactivos, registrando automáticamente los resultados obtenidos.

# Proyecto: Plataforma Web para el Entrenamiento Cerebral y Pausas Activas Universitarias
## 1. Resumen Ejecutivo
Este proyecto propone el desarrollo de una plataforma web optimizada para mitigar la fatiga cognitiva, el estrés académico y mejorar el rendimiento de la comunidad universitaria (estudiantes y docentes). A través de pausas activas mentales basadas en ejercicios interactivos rápidos, el sistema evalúa y estimula la memoria de trabajo, la atención sostenida y la velocidad de reacción psicomotriz. El sistema registrará de forma automática las métricas de rendimiento por sesión, permitiendo a los usuarios monitorear su evolución cognitiva. La solución se encuentra diseñada bajo una arquitectura ligera y simplificada para garantizar un despliegue exitoso en un periodo estricto de dos semanas.
## 2. Definición del Problema
La alta carga académica, los periodos prolongados de evaluación y las extensas jornadas frente a pantallas dentro del entorno universitario generan un decremento medible en la retención de información, la agilidad mental y los niveles de atención de estudiantes y profesores. Actualmente, no se dispone de herramientas institucionales directas, accesibles y rápidas que sirvan para reactivar el flujo cognitivo y medir el desempeño psicomotriz sin interrumpir severamente las actividades diarias.
Para resolver esto, se propone una aplicación web interactiva que ofrece retos cognitivos estandarizados de corta duración (2 minutos). Con el fin de asegurar la viabilidad técnica del desarrollo en un plazo de 2 semanas y respetar estrictamente la privacidad del entorno estudiantil, **la plataforma prescinde por completo del uso de cámaras web o algoritmos complejos de visión artificial**. Toda medición psicomotriz óculo-manual se resolverá midiendo la latencia de respuesta del hardware estándar de interacción humana (mouse y teclado).
## 3. Objetivo General
Desarrollar e implementar una plataforma web funcional de entrenamiento cerebral y pausas activas para el entorno universitario, que permita estimular la agilidad mental y evaluar variables psicomotrices mediante interacciones basadas en teclado y mouse, centralizando los resultados en un historial de progresión personal.
## 4. Objetivos Específicos
 * **Diseñar una interfaz de usuario** limpia, madura y responsiva (estilo Dashboard/Dark-mode) adaptada al perfil tecnológico de estudiantes universitarios.
 * **Implementar tres mecánicas de juego** lógico-interactivas enfocadas en memoria rápida, atención selectiva y velocidad de reacción.
 * **Registrar con precisión** mediante eventos de JavaScript el desempeño psicomotriz (tiempos de respuesta y precisión).
 * **Construir un panel de control** (historial) que consolide las estadísticas históricas de uso del usuario.
## 5. Usuarios y Clientes
### Usuarios
 * **Estudiantes Universitarios:** Quienes acceden a la plataforma para realizar pausas mentales entre clases o sesiones de estudio pesado.
 * **Docentes e Investigadores:** Usuarios que buscan reactivar su enfoque cognitivo durante jornadas prolongadas de planeación o revisión.
### Clientes / Evaluadores
 * Instituciones académicas, facultades de ingeniería/ciencias de la computación, o la cátedra evaluadora del proyecto final.
## 6. Funcionalidades Principales
 * **Autenticación y Registro de Perfiles:** Módulo de acceso básico para la gestión individualizada del histórico de puntuaciones.
 * **Banco de Retos Cognitivos:** Módulo centralizado que almacena la lógica interactiva de las pausas activas.
 * **Restricción Periférica Absoluta (Sin Cámara):** El sistema realiza toda la captura métrica procesando eventos estándar del DOM de JavaScript (click, keydown). No se requiere ni se invoca el uso de flujos de video, lo que minimiza la carga de procesamiento del servidor y simplifica el despliegue del frontend.
 * **Módulo de Analítica (Historial):** Panel visual donde el usuario revisa su evolución temporal en agilidad mental.
## 7. Ejercicios de Estimulación Seleccionados
Para asegurar la entrega en 2 semanas, el alcance se delimita estrictamente a tres ejercicios modulares de alta cohesión lógica:
 1. **Memorama de Retención Rápida:** Ejercicio enfocado en la memoria de trabajo a corto plazo empleando el emparejamiento de matrices de símbolos o términos técnicos basados en un temporizador de degradación visual.
 2. **Secuencia de Colores (Memoria Sostenida):** Reto basado en el patrón clásico de reproducción de secuencias dinámicas incrementales para evaluar la atención y la concentración bajo presión de tiempo.
 3. **Selección del Objeto Correcto (Agilidad Visual y Reacción):** Discriminación de patrones visuales intrusos contra reloj. Este ejercicio mide de forma directa el tiempo de respuesta óculo-manual.
## 8. Indicadores Clave de Desempeño (KPIs)
El backend registrará de forma nativa por cada intento de pausa activa:
 * **Tiempo de Reacción Promedio (ms):** Tiempo transcurrido en milisegundos desde la aparición del estímulo visual hasta el evento de entrada del usuario.
 * **Tasa de Precisión:** Relación porcentual entre aciertos totales y clics totales ejecutados.
 * **Volumen de Errores:** Registro de fallos por sesión para graficar curvas de fatiga.
 * **Frecuencia de Uso:** Registro cronológico de sesiones completadas para evaluar la constancia del usuario.
## 9. Metodología de Desarrollo (Scrum Adaptado)
Se implementa un marco Scrum comprimido en dos ciclos iterativos de desarrollo (Sprints de 1 semana):
 * **Product Owner:** Define el alcance de las funcionalidades y prioriza el backlog enfocado a la comunidad universitaria.
 * **Scrum Master:** Garantiza la eliminación de bloqueos técnicos y el apego estricto al plan de entrega quincenal.
 * **Equipo de Desarrollo (Fullstack):** Encargado de la codificación y despliegue del modelo relacional, lógica de backend y vistas interactivas.
## 10. Planificación Detallada del Cronograma (2 Semanas)
### Sprint 1 (Semana 1) — Cimiento de Arquitectura y Agilidad Visual
 * **Objetivo de Sprint:** Establecer la infraestructura base del sistema y desplegar el primer componente de interacción psicomotriz.
 * **Tareas:**
   * Configuración del entorno Backend en Python utilizando el microframework **Flask**.
   * Diseño e implementación del esquema relacional en **MySQL** (Tablas: usuarios, juegos, puntuaciones).
   * Maquetación de la interfaz base con HTML5 y CSS3 empleando una estética limpia, profesional y optimizada para el entorno web.
   * Desarrollo en JavaScript del juego *Selección del Objeto Correcto* con persistencia de KPIs.
 * **Entregable:** Aplicación base con control de sesiones, menú de navegación principal y un juego operativo que guarda datos en la BD.
### Sprint 2 (Semana 2) — Consolidación Cognitiva y Analítica
 * **Objetivo de Sprint:** Desarrollar los módulos lógicos restantes y unificar la experiencia analítica del usuario.
 * **Tareas:**
   * Desarrollo algorítmico en JavaScript nativo del *Memorama Rápido* y la *Secuencia de Colores*.
   * Construcción del Dashboard de Historial que consume los datos agregados de la BD para mostrar tablas y estadísticas de rendimiento.
   * Ejecución de pruebas funcionales, optimización de la latencia de eventos y depuración de código frontend/backend.
 * **Entregable:** Plataforma integral y funcional al 100%, empaquetada para su presentación y defensa académica.
## 11. Arquitectura Tecnológica
 * **Frontend:** HTML5 (Semántico), CSS3 (Hojas de estilo con variables nativas para estructuración limpia) y JavaScript (ES6+ nativo, manipulación directa del DOM, manejo del estado de los juegos sin frameworks pesados).
 * **Backend:** Python 3.x utilizando Flask para enrutamiento ágil y controladores REST eficientes.
 * **Base de Datos:** MySQL para almacenamiento persistente y transaccional indexado por ID de usuario.
## 12. Beneficios Esperados
 * Incremento en la velocidad de procesamiento de información y capacidad de concentración de los estudiantes.
 * Disponibilidad inmediata y universal desde computadoras institucionales o dispositivos personales sin instalaciones adicionales.
 * Reducción del estrés mediante la desconexión mental estructurada durante los descansos escolares.
 * Validación de datos en tiempo real al poder desplegar el sistema directamente con compañeros de aula para pruebas inmediatas.
## 13. Matriz de Gestión de Riesgos
| Riesgo Identificado | Estrategia de Mitigación Implementada |
|---|---|
| **Complejidad en la captura psicomotriz** | **Exclusión del uso de cámara.** La coordinación motriz se registra evaluando milisegundos de latencia en eventos JavaScript nativos del teclado/mouse, simplificando drásticamente el código. |
| **Tiempo de desarrollo críticamente limitado (2 semanas)** | Reducción estricta del alcance de 5 a 3 juegos. Reutilización de la lógica del temporizador de JavaScript y del despachador de puntuaciones del backend para las tres actividades. |
| **Falta de interés u adopción de los universitarios** | Implementación de un enfoque competitivo mediante el despliegue de récords personales altos dentro del Dashboard principal. |
| **Pérdida de persistencia de datos por desconexión** | Envío de cargas útiles asíncronas vía Fetch API inmediatamente al finalizar cada juego, blindando el registro de KPIs contra cierres accidentales de pestañas. |
